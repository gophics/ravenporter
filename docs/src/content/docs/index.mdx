## What Stays Consistent

<CardGrid stagger>
  <Card title="One Import Vocabulary">
    The Go API and the CLI describe the same pipeline, so scripts, tools, and embedded integrations do not drift apart.
  </Card>
  <Card title="One Internal Representation">
    Models, images, audio, and fonts all land in the same [`ir.Asset`](./ir-overview/), which keeps inspection and downstream tooling consistent.
  </Card>
  <Card title="One Runtime Packaging Story">
    Use [`cache`](./runtime-cache/) once startup time matters more than reparsing source assets on every launch.
  </Card>
</CardGrid>
