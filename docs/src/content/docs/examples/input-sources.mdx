Use this when the asset does not start as a plain path on disk. It shows the same import flow through `os.DirFS`, raw bytes, and a `ReadSeekerAt`-compatible reader such as `*bytes.Reader`.

## Key APIs

- `ravenporter.ImportFS()`
- `ravenporter.ImportBytes()`
- `ravenporter.ImportReader()`

## Run

```bash
go run ./examples/input-sources
```

## Expected Output

```text
dirfs=obj/1
bytes=obj/1
reader=obj/1
```
