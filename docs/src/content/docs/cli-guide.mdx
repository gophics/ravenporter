## Install

Requires Go `1.25.7` or newer.

```bash
go install github.com/gophics/ravenporter/cmd/ravenporter@v0.1.0
```

## Main Commands

| Command | Purpose |
| --- | --- |
| `import` | Import source assets and emit JSON IR |
| `export` | Export one asset to a target emitter format |
| `batch` | Walk a directory and emit JSON IR for supported files |
| `convert` | Shorthand for import plus export, with the output format inferred from `--out` |
| `cook` | Import and write a `.rpcache` runtime asset |
| `info` | Print high-level asset information |
| `inspect` | Print a deeper asset breakdown |
| `validate` | Run direct validation on source assets |
| `formats` | List built-in import formats |
| `steps` | List canonical process-step names |
| `profile export` | Save the effective import config as TOML |

## Common Workflows

### Import To JSON IR

```bash
ravenporter import assets/scene.glb --out out --pretty
```

### Export JSON Explicitly

```bash
ravenporter export assets/scene.glb --format json --out out/scene.json --pretty
```

### Convert Using The Output Extension

```bash
ravenporter convert assets/scene.glb --out out/scene.json --pretty
```

`convert` uses the same emitter path as `export`, but it infers the format from `--out` instead of taking `--format` directly. Today that still means JSON.

### Cook A Runtime Cache

```bash
ravenporter cook assets/scene.glb --out build/scene.rpcache --embed-textures
```

### Inspect An Asset

```bash
ravenporter info assets/scene.glb
ravenporter inspect assets/scene.glb
```

`info` and `inspect` both work on cooked `.rpcache` files. Use `info --json` for a compact scripted summary, or `inspect --json` when you want the full asset payload.

### Validate A Source Asset

```bash
ravenporter validate assets/scene.glb
ravenporter validate assets/scene.glb --json
```

`validate` currently runs mesh, node, material, texture-reference, and animation-oriented checks, so it is most informative on scene and model assets.

### Batch A Directory

```bash
ravenporter batch assets --out out --pretty --recursive
```

`batch` stays at the top level unless you pass `--recursive`.

## Profiles

Export a reusable TOML profile:

```bash
ravenporter profile export --preset quality --embed-textures --scale 0.01 --out profiles/game.toml
```

Reuse it on a later command:

```bash
ravenporter import assets/scene.fbx --profile-file profiles/game.toml --out out
```

Explicit flags are assembled after `--profile-file`, so direct flags override the loaded profile.

## Process Step Overrides

List the canonical step names:

```bash
ravenporter steps
```

Enable or disable them directly:

```bash
ravenporter cook assets/scene.glb --enable-step embed-textures --disable-step find-instances
```

## Useful Flags

- `--preset fast|quality|max-quality`
- `--profile-file path/to/profile.toml`
- `--embed-textures`
- `--scale 0.01`
- `--up-axis Y|Z`
- `--decode-max-file-size`
- `--decode-max-vertices`
- `--enable-step`
- `--disable-step`
- `--report-file`
- `--cache-image-pixels never|if-present|always`
- `--cache-max-embedded-media-bytes`

## Command Mapping Back To The Library

- `import`, `export`, `batch`, and `convert` all start from the root import API.
- `cook` adds [`cache.Write()`](../cache-package/) after import.
- `validate` uses [`detect`](../detect-package/) and [`validate`](../validate-package/) directly on source assets, with the current rule set focused on scene and model content.
- `profile export` uses [`ResolveProfile()`](../profiles-and-presets/).
