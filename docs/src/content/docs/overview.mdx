## What RavenPorter Gives You

- One import surface for models, images, audio, and fonts.
- A flat, index-based IR that is easy to traverse, transform, serialize, and validate.
- A strict runtime cache format through [`cache`](../runtime-cache/) for faster load paths.
- A small root package API with optional lower-level hooks in [`detect`](../detect-package/), [`process`](../process-package/), [`validate`](../validate-package/), and [`emit/json`](../emit-json-package/).

If you want the design rationale behind those choices, see [Why RavenPorter](../why-ravenporter/).

## Main Layers

| Layer | Purpose |
| --- | --- |
| [`ravenporter`](../ravenporter-package/) | Public import entrypoint, option model, profile helpers, supported-format discovery |
| [`cache`](../cache-package/) | Cook and load `.rpcache` runtime assets |
| [`detect`](../detect-package/) | Registry and decoder contracts for custom format detection and decode |
| [`process`](../process-package/) | Built-in post-import processing catalog |
| [`validate`](../validate-package/) | Structural and semantic validation passes |
| [`ir`](../ir-asset-and-graph/) | Runtime-oriented asset representation |

## Library Vs CLI Vs Cache

- Reach for the library when RavenPorter is part of a larger application or build pipeline.
- The CLI is the better fit for inspection, conversion, validation, or cooking from the shell.
- Bring in the cache package when startup time matters more than reparsing source assets on every launch.

## Typical Flow

1. Import source content with `ravenporter.ImportPath()` or another root-package entrypoint.
2. Inspect the returned `Result` and `Report`.
3. Optionally persist the effective behavior as a TOML profile.
4. Cook the imported result with `cache.Write()` if you want a runtime-ready artifact.
5. Load the cooked asset later with `cache.Open()`, `cache.OpenFS()`, or `cache.Read()`.

## Current Scope And Limits

- The public API stays centered on the root package instead of exposing internal pipeline packages directly.
- The JSON emitter is for inspection and tooling handoff, not for final runtime packaging.
- Draco- and meshopt-compressed glTF meshes are decoded during import before optional runtime cache cooking.

For the design tradeoffs, continue with [Why RavenPorter](../why-ravenporter/). For code first, go to [Quick Start](../quick-start/). If the representation matters most, open [IR Overview](../ir-overview/).
