  The current cache container version in this repository is `1`. Older cooked caches from earlier commits should be treated as disposable and fully rebuilt when the scene serializer layout changes, because this branch does not keep compatibility shims for older cache payloads.
</Aside>

<Aside type="caution" title="Cook only self-contained runtime assets">
  cache.Write() rejects external texture references. If you need a single-file runtime package, import with WithEmbedTextures() before cooking.
</Aside>

## Write A Cache File

```go
package main

import (
	"bytes"
	"context"
	"log"

	"github.com/gophics/ravenporter"
	"github.com/gophics/ravenporter/cache"
)

func main() {
	result, err := ravenporter.ImportPath(
		context.Background(),
		"assets/scene.glb",
		ravenporter.WithEmbedTextures(),
	)
	if err != nil {
		log.Fatal(err)
	}

	var cooked bytes.Buffer
	if err := cache.Write(&cooked, result); err != nil {
		log.Fatal(err)
	}

	pkg, err := cache.Read(bytes.NewReader(cooked.Bytes()), int64(cooked.Len()))
	if err != nil {
		log.Fatal(err)
	}
	defer pkg.Close()
}
```

## Reading

| Function | Best fit |
| --- | --- |
| `cache.Open(path, options...)` | The cooked asset lives on disk |
| `cache.OpenFS(fsys, path, options...)` | The cooked asset lives inside an `fs.FS` |
| `cache.Read(readerAt, size, options...)` | You already have an `io.ReaderAt` and the asset size |

Call `(*cache.Asset).Close()` when you open a lazy reader-backed cache asset.

## Lazy Media Model

By default:

- `MANF` and `SCEN` data are loaded eagerly.
- `BLOB` payloads for images, audio, and vector-font raw bytes stay reader-backed.
- `DecodePixels()` and `DecodeSamples()` still work after opening a cooked asset because the cache layer restores decode callbacks for supported formats.

Use `cache.WithEagerMedia()` if you want all embedded media materialized during `Open()` or `Read()`.

## Write Options

| Option | Purpose |
| --- | --- |
| `cache.WithImagePixels(mode)` | Control when decoded image pixels are baked into the cache |
| `cache.WithMaxEmbeddedMediaBytes(limit)` | Reject oversized cooked assets |

Supported pixel modes:

- `cache.ImagePixelsNever`
- `cache.ImagePixelsIfPresent`
- `cache.ImagePixelsAlways`

## Strict Validation

`cache.Write()` validates the import result before serialization.

It will reject assets when:

- the `Result` or `Asset` is `nil`
- a texture still points at an external image file path
- a material contains unsupported arbitrary property types

For single-file runtime packages, use `WithEmbedTextures()` during import so texture dependencies are embedded before cooking.

## Cache Manifest

Each cooked file includes a manifest with:

- cache format version
- source format
- source profile
- dependencies
- provenance notes
- asset summary counts

See [Cache Format Model](../cache-format-model/) for the container layout and chunk model.
