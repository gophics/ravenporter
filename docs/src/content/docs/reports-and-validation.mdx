<Aside type="note" title="Result handling">
  Treat the error as authoritative, then inspect the result when it is non-nil. That gives you the strict failure signal and the richest available diagnostics.
</Aside>

## Report Shape

| Field | Meaning |
| --- | --- |
| `Source` | Input name, path, detected format, source metadata, preset, profile file, effective options, and provenance notes |
| `Issues` | Stable findings with `stage`, `severity`, `code`, and `message` |
| `Dependencies` | Direct external dependencies discovered during decode or asset collection |
| `Summary` | High-level counts for scenes, meshes, materials, nodes, images, audio, fonts, and more |

## Pipeline Stages

The root package uses these stable stage names in `Report.Issues`:

| Stage | Meaning |
| --- | --- |
| `detect` | Format detection and decoder lookup |
| `decode` | Decoder execution |
| `validate-structural` | Integrity checks before processing |
| `process` | Built-in post-processing |
| `validate-semantic` | Logical checks after processing and load-mask pruning |

## Severity

| Severity | Meaning |
| --- | --- |
| `error` | A fatal or serious issue |
| `warning` | Non-fatal issue worth surfacing |
| `info` | Informational note |

## Important Behavior

- Structural validation errors stop the import and return an error.
- Semantic validation findings are appended to the report after processing. The import can still return a usable `Result` with semantic errors recorded in `Report.Issues`.
- Decoder and pipeline failures can still leave you with a partial `Result`, especially when RavenPorter already knows the source format and has report context to preserve.
- Dependencies are collected from both decoder-reported provenance and asset-level references such as external image paths.

## Example

```json
{
  "source": {
    "input_name": "scene.glb",
    "detected_format": "glb",
    "preset": "quality"
  },
  "issues": [
    {
      "stage": "validate-semantic",
      "severity": "warning",
      "code": "DEGENERATE_TRIANGLE",
      "message": "mesh[0].prim[0] triangle 12 has duplicate indices"
    }
  ],
  "summary": {
    "scenes": 1,
    "meshes": 4,
    "materials": 3,
    "textures": 5
  }
}
```

## Standalone Validation API

Open [`validate`](../validate-package/) when you want direct control over validation for scene and model assets:

```go
result := validate.Asset(asset)
if !result.OK() {
	for _, issue := range result.Errors {
		log.Printf("error %s: %s", issue.Code, issue.Message)
	}
}
```

The current standalone rules focus on mesh, node, material, texture-reference, and animation checks. Non-scene assets usually rely more on import-time decode failures than on additional `validate` findings.
